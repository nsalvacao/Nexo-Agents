Architectural Decision: CLI-Native Pattern

Rationale

  • Stateless execution: No backend services required
  • Token efficiency: 50-70% reduction via focused prompts
  • Composability: Commands chainable via shell operators
  • Discovery automation: Native ~/.gemini/commands/ loading
  • Shell integration: Context-aware via !{command}
